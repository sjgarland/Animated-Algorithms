When a problem cannot be solved by an explicit computation, it can often be solved by a search: finding a telephone number involves searching through a list of names; finding the prime factors of an integer involves searching for divisors; finding the exit from a maze involves searching for a path to the exit.

Searches may employ brute force (for example, beginning with the first name in a list and proceeding serially). Or they may capitalize on the organization of the list to search in a less time-consuming fashion (for example, by starting at a name's approximate location in a list and proceeding from there). When all else fails, a search may resort to trial and error; even then, it's best to search intelligently rather than haphazardly, as when trying to find a path out of a maze, and to backtrack and try again when a search fails.

The goal of a simple search is to determine whether an item x occurs in a list a[1], ..., a[n] of items. If x is in the list, it may be useful to identify where by producing an index i such that x = a[i]; this index i indicates, for example, which element a[i] in a list of names must be modified to correct a misspelling or which element b[i] in another list b[1], ..., b[n] contains the telephone number of a[i].

More generally, the goal of a search is to determine whether a list contains an item with a particular property (for example, whether a list of words contains one beginning with the letter X) and, if it does, to identify where the item can be found.

Search algorithms

The first two search algorithms look for items in simple lists. One is suited to lists whose elements occur in no particular order (for example, a shuffled deck of cards), the other to lists arranged in some specific order (for example, an alphabetized list of names). The other algorithms conduct searches when information is not so neatly arranged in a list.

  • Linear search: find (or insert) an element in an unordered list
  • Binary search: find (or insert) an element in an ordered list
  • Maze searches
    • Randomized search for an exit
    • Backtracking search
    • Hand-on-the-wall algorithm
